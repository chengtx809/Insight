# 吴军笔风教学文章生成器 - 通用提示词模板

## 使用说明

这个提示词模板可以生成类似吴军《数学之美》笔风的深度教学文章。适用于任何知识点、算法、编程概念的讲解。

**使用方法：**
1. 将 `{知识点主题}` 替换为你要讲解的具体内容
2. 将 `{相关题目列表}` 替换为具体题目信息
3. 将 `{目标受众}` 替换为读者定位
4. 直接使用这个提示词，让AI生成文章

# 核心要求

## 1. 写作风格要求

**吴军笔风的核心特征：**
- **从第一性原理出发**：不满足于"怎么做"，而要追问"为什么"
- **历史与现代交织**：善用历史故事、科学家轶事引入概念
- **哲学与技术融合**：技术背后有思想，算法背后有智慧
- **类比化抽象**：用生活化的类比解释抽象概念
- **层次分明递进**：从简单到复杂，从具体到抽象
- **注重思维方式**：不是记住"术"，而是理解"道"

**语言特点：**
- 口语化但有深度，像在和朋友聊天
- 善用反问、设问引发思考
- 适当引用名人名言，但不堆砌
- 段落短小精悍，节奏明快
- 代码和文字有机结合

## 2. 文章结构要求

### 引子（约800-1000字）

**必须包含：**
1. 一个历史故事/科学家轶事/古代智慧
   - 优先选择：古希腊数学家、文艺复兴科学家、计算机先驱
   - 要求：真实、有趣、与主题相关

2. 从故事引出核心概念
   - 用"这个故事揭示了..."过渡
   - 提出文章要回答的核心问题

3. 点题：说明文章的目标
   - "这篇文章将讲述..."
   - "当你理解了X，你就理解了Y"

**示例开头：**
> "公元前300年，欧几里得在《几何原本》中证明了一个看似简单的命题：任何合数都可以分解为质数的乘积。两千多年后，当计算机科学家们试图..."

### 主体章节（3-5章，每章1500-2500字）

**章节递进逻辑：**
1. **第一章**：最基础的概念
   - 从零开始，不假设任何背景知识
   - 用大量类比和可视化
   - 建立直觉理解

2. **第二章**：核心机制
   - 深入技术细节
   - 讲解算法/原理
   - 配合代码示例

3. **第三章**：难点突破
   - 解决学习中的常见困惑
   - 用对比法突出关键差异
   - 提供"顿悟时刻"

4. **第四章**：应用与扩展
   - 实际案例分析
   - 与其他知识的联系
   - 更深层的哲学思考

5. **第五章**（可选）：方法论
   - 如何学习这个知识
   - 如何迁移到其他领域
   - 建立知识体系

**每章结构：**
```
## 第X章：章节标题（要有诗意和吸引力）

### 小节1：用问题引入
提出一个引人思考的问题

### 小节2：概念讲解
- 定义（简洁、准确）
- 类比（生动、贴切）
- 可视化（图、表、代码）

### 小节3：深入理解
- 为什么这样设计？
- 有什么替代方案？
- 背后的权衡是什么？

### 小节4：常见误区
- 初学者容易犯的错误
- 为什么会这样想？
- 如何纠正？
```

### 结语（约600-800字）

**必须包含：**
1. 升华主题：从技术到思想
   - "X不仅是一个技术，更是一种思维方式"

2. 知识迁移：联系更广阔的领域
   - "这种思想在Y领域也适用"

3. 学习建议：具体可操作
   - 推荐练习题
   - 推荐阅读资料
   - 下一步学习方向

4. 激励性收尾
   - 引用名言或提出愿景
   - 鼓励读者继续探索

## 3. 内容深度要求

### 三个层次的融合

**第一层：技术层（What & How）**
- 准确的定义和概念
- 清晰的示例
- 正确的步骤

**第二层：原理层（Why）**
- 为什么这样设计？
- 背后的数学/逻辑基础是什么？
- 有什么替代方案？为什么选择这个？

**第三层：哲学层（Essence）**
- 这个技术反映了什么思维方式？
- 与人类认知/历史发展有什么联系？
- 对其他领域有什么启发？

### 具体要求

**必须包含的元素：**

1. **历史溯源**（至少1-2处）
   - 谁最早提出这个概念？
   - 为什么在那个时代提出？
   - 如何演变到今天？

2. **第一性原理分析**（核心章节）
   - 这个概念最基本的假设是什么？
   - 从这些假设能推导出什么？
   - 哪些是本质，哪些是实现细节？

3. **类比与比喻**（至少3-5个）
   - 用生活化场景类比抽象概念
   - 类比要准确，不能误导
   - 示例："递归就像俄罗斯套娃"

4. **对比分析**（至少2-3组）
   - X vs Y有什么不同？
   - 何时用X，何时用Y？
   - 列表或表格对比

5. **可视化说明**（题目、代码、图表、示例）
   - 每个核心概念都要有示例
   - 复杂过程要画图或列步骤
   - 用注释解释关键步骤

6. **常见误区**（至少3-5个）
   - 初学者容易犯的错误
   - 为什么会这样想？（认知原因）
   - 如何纠正？（正确思路）

7. **名人名言**（2-4个，自然嵌入）
   - 数学家、科学家、哲学家的相关论述
   - 要真实可考，不要编造
   - 用于点题或升华

## 5. 质量标准

**文章必须达到的标准：**

1. **深度足够**
   - 单篇不少于5000字
   - 不能只停留在"怎么做"，必须解释"为什么"
   - 要有哲学层面的思考

2. **生动有趣**
   - 有历史故事、科学家轶事
   - 有生活化的类比
   - 语言口语化，避免教科书式的枯燥

3. **逻辑严密**
   - 概念定义准确
   - 答案示例正确
   - 推理过程清晰

4. **层次分明**
   - 从简单到复杂
   - 从具体到抽象
   - 每章都有明确的主题

5. **实用性强**
   - 有具体的答案示例
   - 有可操作的练习建议
   - 有学习资源推荐

## 6. 避免的陷阱

**不要：**
1. ❌ 直接堆砌定义和公式
2. ❌ 假设读者有很多背景知识
3. ❌ 只讲"怎么做"，不讲"为什么"
4. ❌ 使用过于学术化的语言
5. ❌ 堆砌名人名言（每处引用都要有意义）
6. ❌ 编造历史故事或名言
7. ❌ 答案没有注释和解释
8. ❌ 忽视初学者的困惑点

**要：**
1. ✅ 用故事和类比引入概念
2. ✅ 从第一性原理出发讲解
3. ✅ 代码/过程/数学推理/物理推理/其他推理和文字有机结合
4. ✅ 讨论设计权衡和替代方案
5. ✅ 预设并回答读者的疑问
6. ✅ 升华到思维方式和哲学层面
7. ✅ 提供具体可操作的建议
